## サービス概要
外食時に「何を食べるか迷う」課題を解決するためのWebアプリです。  
気分・位置情報をもとに、最適な飲食店を提案します。

---

## 開発背景
外食時、特に土地勘のない場所では評価の高い店舗が多く、比較に時間がかかるという課題があります。  
実際に出張先で店選びに30分以上迷った経験から、「評価」ではなく「今の自分に合う店」を提案する仕組みを開発しました。

---

## ユーザー層
- 20〜30代の会社員  
- 出張・転勤などで土地勘のない場所で外食する人  
- 限られた時間で満足できる食事を選びたい人 

---

## 特徴・差別化

### 状況適応型レコメンド
- 気分・位置情報をもとに店舗を提案  
- 「評価の高い店」ではなく「今の自分に合う店」を提案

### 従来サービスとの違い
- 食べログ：評価や口コミを中心とした検索で、ユーザーのその時の状態までは考慮されない  
- Googleマップ：位置情報ベースの検索が中心で、ユーザーの気分や状況は反映されにくい 
- 本サービス：気分 × 位置情報を組み合わせ、「今の自分に最適な店」を提案 

---

## 主な機能

### MVP（実装済み）
- ユーザー登録 / ログイン  
- 位置情報取得  
- 気分選択  
- 周辺店舗検索（Google Places API (New)）

### 今後の拡張（未実装）
- お気に入り登録  
- X連携による気分推定  
- 天候データ連携  
- 履歴ベースのレコメンド最適化  

---

## 実装のポイント
- 気分ごとに検索キーワード・評価条件を動的に変更（フロントでロジック管理）
- Google Places API（New）の searchByText を使用した店舗検索（キーワードベースで柔軟な検索条件を構築できるため採用）
- 「気分選択 → 検索条件生成 → API取得 → フィルタリング → 表示」のフローで構成
- APIレスポンスを正規化し、フロントで扱いやすいデータ構造に変換
- 距離計算（Haversine）によるフィルタリングを実装（現在地からの距離で絞り込み）
- 評価・距離の複合条件による独自の絞り込みロジックを実装
- 検索ロジックをフロント側に持たせることで、ユーザー操作に対する即時反映とUX向上を実現
- Google Cloudの予算アラートとAPIキー制限（HTTPリファラー）を設定し、過剰課金を防止
- Places APIの403エラー（Billing未設定）に対応し、エラーハンドリングを実装
